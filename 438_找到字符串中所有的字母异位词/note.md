# 力扣 438. 找到字符串中所有字母异位词 学习笔记

## 题目描述
给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的异位词子串，返回这些子串的起始索引。输出顺序不限。

### 异位词的含义
如果两个字符串包含的字符种类相同，并且每种字符出现次数也完全相同，只是顺序不同，那么它们就是异位词。

例如：
- `"abc"` 和 `"cba"` 是异位词
- `"ab"` 和 `"ba"` 是异位词
- `"ab"` 和 `"aa"` 不是异位词

---

## 题目本质
这道题的关键是：在字符串 `s` 中，找出所有长度等于 `p.length` 的子串，判断它们是否与 `p` 具有相同的字符频次分布。因为“异位词”只关心有哪些字符以及每个字符出现多少次，而不关心字符顺序。

---

## 为什么适合用滑动窗口
假设 `p` 的长度是 `m`。那么在 `s` 中，任何可能成为答案的子串，长度都必须是 `m`。所以我们可以维护一个长度固定为 `m` 的窗口，让这个窗口在 `s` 上从左到右滑动，判断每个窗口里的字符频次是否与 `p` 相同。这就是典型的 **固定长度滑动窗口** 问题。

---

## 核心思路

### 统计 `p` 的字符频次
用一个长度为 26 的数组 `need[26]` 记录 `p` 中每个小写字母的出现次数。

### 维护当前窗口的字符频次
再用一个长度为 26 的数组 `window[26]` 记录当前窗口中的字符频次。

### 窗口滑动
先让窗口覆盖 `s` 的前 `pLen` 个字符。之后每向右滑动一步：
* 新字符进入窗口：计数加 1
* 旧字符离开窗口：计数减 1
如果某一时刻 `window` 与 `need` 完全相同，说明当前窗口就是 `p` 的一个异位词。

---

## C 语言实现

```c
#include <stdlib.h>
#include <string.h>

static int isSame(int* a, int* b) {
    for (int i = 0; i < 26; i++) {
        if (a[i] != b[i]) {
            return 0;
        }
    }
    return 1;
}

int* findAnagrams(char* s, char* p, int* returnSize) {
    int sLen = strlen(s);
    int pLen = strlen(p);

    *returnSize = 0;

    if (sLen < pLen) {
        return NULL;
    }

    int need[26] = {0};
    int window[26] = {0};

    // 统计 p 和初始窗口的频次
    for (int i = 0; i < pLen; i++) {
        need[p[i] - 'a']++;
        window[s[i] - 'a']++;
    }

    int* result = (int*)malloc(sizeof(int) * sLen);

    // 判断初始窗口
    if (isSame(need, window)) {
        result[(*returnSize)++] = 0;
    }

    // 滑动窗口
    for (int i = pLen; i < sLen; i++) {
        window[s[i] - 'a']++;          // 新字符进入
        window[s[i - pLen] - 'a']--;   // 旧字符移除

        if (isSame(need, window)) {
            result[(*returnSize)++] = i - pLen + 1;
        }
    }

    return result;
}
```

---

## 代码逐步解释

### `isSame` 函数
作用是比较两个长度为 26 的数组是否完全相同。因为字符集只有 26 个小写字母，比较数组的代价是固定的常数。

### 边界判断
如果 `s` 比 `p` 还短，不可能存在匹配的窗口，直接返回 `NULL`。

### 统计 `need` 和初始 `window`
这一步同步统计模式串 `p` 的频次和 `s` 中第一个窗口的频次。

### 滑动窗口逻辑
窗口右移一格时，`s[i]` 进入，`s[i - pLen]` 离开。更新频次数组后立即调用 `isSame` 判断。

---

## 复杂度分析

### 时间复杂度
* **初始化**：$O(pLen)$
* **滑动过程**：窗口滑动约 $n$ 次，每次更新为 $O(1)$，比较数组为 $O(26)$。
* **总计**：$O(n)$，其中 $n$ 为 `s` 的长度。

### 空间复杂度
使用两个固定长度为 26 的数组，额外空间复杂度为 **$O(1)$**。

---

## 易错点总结
* **窗口长度不固定**：本题窗口必须始终等于 `p.length`。
* **忘记处理初始窗口**：第一个窗口 `[0, pLen-1]` 需要在循环开始前先判断。
* **更新顺序**：滑动时必须保证“一进一出”的逻辑准确，否则频次会漂移。
* **C 语言细节**：返回数组需要手动 `malloc`，且必须正确更新 `returnSize`。
