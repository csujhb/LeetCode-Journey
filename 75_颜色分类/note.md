# 颜色分类 (LeetCode 75) 核心学习笔记

## 计数排序法（两次扫描）

**思路剖析：**
这是最直观的基础解法。因为题目明确说明数组中只包含 `0`、`1`、`2` 这三种特定元素，我们可以直接遍历一遍数组，分别统计这三种颜色出现的总次数。统计完成之后，根据计数结果进行第二次遍历，按照红（`0`）、白（`1`）、蓝（`2`）的顺序依次重写并覆盖原数组。虽然需要扫描两次，但整体时间复杂度依然维持在线性级别。

**C语言实现代码：**
```c
void sortColors(int* nums, int numsSize) {
    int count0 = 0;
    int count1 = 0;
    int count2 = 0;
    
    // 第一趟扫描：统计各个元素的出现次数
    for (int i = 0; i < numsSize; i++) {
        if (nums[i] == 0) {
            count0++;
        } else if (nums[i] == 1) {
            count1++;
        } else {
            count2++;
        }
    }
    
    // 第二趟扫描：根据计数结果重写数组
    int idx = 0;
    while (count0--) {
        nums[idx++] = 0;
    }
    while (count1--) {
        nums[idx++] = 1;
    }
    while (count2--) {
        nums[idx++] = 2;
    }
}
```

## 三指针法 / 荷兰国旗算法（一次扫描）

**思路剖析：**
这是满足题目进阶要求的绝对最优解，由计算机科学家 Dijkstra 提出。核心思想是利用三个指针，在遍历过程中动态地将数组划分为三个纯色区域。我们设置左指针 `p0` 指向下一个 `0` 应该存放的位置，右指针 `p2` 指向下一个 `2` 应该存放的位置，以及一个当前遍历指针 `i`。

在遍历指针 `i` 尚未越过右指针 `p2` 的前提下，不断判定当前元素的值：
* **遇到 `0`：** 该元素属于最左侧区域。将其与 `p0` 指向的元素交换。交换后，左侧区域扩大，因此 `p0` 向右移动；同时当前位置处理完毕，`i` 向右移动。
* **遇到 `1`：** 该元素属于中间区域，位置正确，无需任何交换操作，直接将 `i` 向右移动。
* **遇到 `2`：** 该元素属于最右侧区域。将其与 `p2` 指向的元素交换。交换后，右侧区域扩大，因此 `p2` 向左移动。**关键细节：** 此时 `i` 保持不动，因为从右侧换过来的新元素还没有经过检查，必须在下一次循环中继续判断它究竟是 `0`、`1` 还是 `2`。

**C语言实现代码：**

```c
void sortColors(int* nums, int numsSize) {
    int p0 = 0;
    int p2 = numsSize - 1;
    int i = 0;

    while (i <= p2) {
        if (nums[i] == 0) {
            // 遇到 0，换到左边去，双方指针推进
            int temp = nums[i];
            nums[i] = nums[p0];
            nums[p0] = temp;
            p0++;
            i++;
        } 
        else if (nums[i] == 2) {
            // 遇到 2，换到右边去，右边界收缩
            // 注意：i 不动，因为新换过来的 nums[i] 还需要下一轮重新判断
            int temp = nums[i];
            nums[i] = nums[p2];
            nums[p2] = temp;
            p2--;
        } 
        else {
            // 遇到 1，跳过即可
            i++;
        }
    }
}
```

## 核心方法总结与对比

| 算法方法 | 时间复杂度 | 空间复杂度 | 核心特点与适用场景 |
| :--- | :--- | :--- | :--- |
| **计数排序法** | $O(n)$ | $O(1)$ | **特点：** 思路极其简单，代码不易出错，非常符合直觉。<br>**场景：** 适合作为快速 AC 的保底方案。缺点是需要遍历两次，且包含大量无意义的重写操作。 |
| **三指针法** | $O(n)$ | $O(1)$ | **特点：** 完美的进阶解法，真正的单次扫描。通过巧妙的指针移动与数值交换，省去了重写时的开销。<br>**场景：** 面试中最受青睐的解法，体现了对双指针思想及边界条件控制的深刻理解。 |
