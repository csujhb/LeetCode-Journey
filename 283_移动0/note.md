# 力扣 283. 移动零（Move Zeroes）学习笔记

## 题目描述

给定一个数组 `nums`，将所有 `0` 移动到数组末尾，同时保持非零元素的相对顺序不变。

要求：

* 必须 **原地修改** 数组
* 不能额外复制一个新数组

---

## 题目本质

这道题本质上是在做两件事：

1.  把所有 **非零元素按原顺序挪到前面**
2.  把剩下的位置全部补成 `0`

重点不在“移动 0”，而在于：

* **保留非零元素相对顺序**
* **原地操作**
* **尽量高效**

---

## 核心思路：双指针 / 快慢指针

定义两个变量：

* `i`：扫描指针，用来遍历整个数组
* `k`：写入指针，表示“下一个非零元素应该放的位置”

遍历数组时：

* 如果 `nums[i] != 0`，就把它放到 `nums[k]`
* 然后 `k++`

遍历结束后：

* 下标 `[0, k-1]` 位置上都是非零元素
* 下标 `[k, numsSize-1]` 位置上全部补 `0`

---

## 为什么这样做是对的

因为我们是 **从左到右** 扫描数组的：

* 遇到的非零元素，按出现顺序依次写入前面
* 所以非零元素的 **相对顺序不会改变**

例如：

```text
原数组： [0, 1, 0, 3, 12]
非零元素出现顺序：1, 3, 12
整理后： [1, 3, 12, 0, 0]
```


---

## C 语言实现

### 覆盖法（推荐）

```c
void moveZeroes(int* nums, int numsSize) {
    int k = 0;  // 下一个非零元素该放的位置

    // 先把所有非零元素按顺序放到前面
    for (int i = 0; i < numsSize; i++) {
        if (nums[i] != 0) {
            nums[k++] = nums[i];
        }
    }

    // 剩余位置补 0
    while (k < numsSize) {
        nums[k++] = 0;
    }
}
```

---

## 代码逐句解释

### 函数定义

```c
void moveZeroes(int* nums, int numsSize)
```

参数含义：

* `nums`：整型数组首地址
* `numsSize`：数组长度

返回值：

* `void`，因为题目要求直接在原数组上修改，不需要返回新数组

---

### 变量定义

```c
int k = 0;
```

`k` 表示：

* 当前已经放好了多少个非零元素
* 或者说，下一个非零元素应该写入的位置

---

### 第一轮遍历：收集非零元素

```c
for (int i = 0; i < numsSize; i++) {
    if (nums[i] != 0) {
        nums[k++] = nums[i];
    }
}
```

含义：

* `i` 从左到右遍历整个数组
* 遇到非零元素，就复制到前面
* `k` 始终指向下一个该写的位置

---

### 第二轮遍历：补零

```c
while (k < numsSize) {
    nums[k++] = 0;
}
```

因为前面已经放好了所有非零元素，所以剩余位置都应该填 `0`。

---

## 例子推演

### 示例 1

输入：

```text
nums = [0, 1, 0, 3, 12]
```

**第一步：提取非零元素**

遍历过程中：

* `i = 0`，`nums[0] = 0`，跳过
* `i = 1`，`nums[1] = 1`，写入 `nums[0]`
* `i = 2`，`nums[2] = 0`，跳过
* `i = 3`，`nums[3] = 3`，写入 `nums[1]`
* `i = 4`，`nums[4] = 12`，写入 `nums[2]`

此时数组前半部分逻辑上变成：

```text
[1, 3, 12, ?, ?]
```

**第二步：补 0**

```text
[1, 3, 12, 0, 0]
```

---

### 示例 2

输入：

```text
nums = [1, 2, 3]
```

第一轮遍历后：

```text
[1, 2, 3]
```

因为没有 0，所以第二轮不需要补 0。

输出：

```text
[1, 2, 3]
```

---

### 示例 3

输入：

```text
nums = [0, 0, 0]
```

第一轮遍历：

* 没有非零元素
* `k = 0`

第二轮遍历：

* 全部补 0

输出：

```text
[0, 0, 0]
```

## 时间复杂度与空间复杂度

### 时间复杂度

* 第一轮遍历：$O(n)$
* 第二轮补零：$O(n)$

总时间复杂度：

```text
O(n)
```

---

### 空间复杂度

只使用了少量变量 `i` 和 `k`，没有开新数组：

```text
O(1)
```

---

## 为什么不能直接删除 0 再补到后面

因为题目要求：

* **原地操作**
* 保持非零元素顺序
* 尽量高效

如果每遇到一个 `0` 都把后面元素整体前移，会导致频繁移动数据，效率较低，最坏可能达到：

```text
O(n^2)
```

而双指针方法只需要线性遍历，效率更高。

---

## 另一种写法：交换法

也可以使用交换法：

```c
void moveZeroes(int* nums, int numsSize) {
    int k = 0;

    for (int i = 0; i < numsSize; i++) {
        if (nums[i] != 0) {
            int temp = nums[i];
            nums[i] = nums[k];
            nums[k] = temp;
            k++;
        }
    }
}
```

### 思路

* `k` 指向当前应该放非零元素的位置
* `i` 向后扫描
* 遇到非零元素就和 `nums[k]` 交换

### 示例

```text
原数组： [0, 1, 0, 3, 12]

i=1 时，将 1 与 nums[0] 交换 -> [1, 0, 0, 3, 12]
i=3 时，将 3 与 nums[1] 交换 -> [1, 3, 0, 0, 12]
i=4 时，将 12 与 nums[2] 交换 -> [1, 3, 12, 0, 0]
```

---

## 两种方法对比

| 方法 | 思路 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **覆盖法** | 先收集非零元素，再补 0 | 简洁清晰，容易理解 | 有些元素可能被重复赋值 |
| **交换法** | 遇到非零元素就交换到前面 | 一次遍历完成 | 对初学者稍难 |

一般推荐优先掌握 **覆盖法**。

---

## 易错点总结

### 易错点 1：误以为第一个元素必须是 0

不是。算法不依赖第一个元素是不是 0，它只依赖：

* 扫描所有元素
* 收集非零元素到前面

例如：

```text
[1, 0, 2, 0, 3] -> [1, 2, 3, 0, 0]
```

同样正确。

---

### 易错点 2：忘记补 0

只把非零元素移到前面还不够，后面必须显式补 `0`。

错误示意：

```text
原数组： [0, 1, 0, 3, 12]
只做第一步后，数组前面可能是 [1, 3, 12, ...]
但后面位置不一定自动变为 0
```

---

### 易错点 3：破坏相对顺序

如果用不恰当的交换方式，可能会改变非零元素原有顺序。
本题要求 **保持顺序不变**，因此必须谨慎设计移动方式。

---

## 这题和哪些题很像

这题和下面这些题思想非常接近：

* 删除数组中的某个元素
* 删除有序数组中的重复项
* 快慢指针覆盖写入类问题

共同点都是：

* 一个指针负责扫描
* 一个指针负责写入有效元素

因此，这题是典型的 **数组原地覆盖 + 双指针** 模板题。
