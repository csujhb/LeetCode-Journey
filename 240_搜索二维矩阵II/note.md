# 搜索二维矩阵 II 学习笔记

---

## 题目描述

给定一个 $m \times n$ 的整数矩阵 `matrix`，以及一个目标值 `target`。

矩阵具有以下性质：
* 每一行从左到右升序排序
* 每一列从上到下升序排序

要求判断 `target` 是否存在于矩阵中，存在返回 `true`，否则返回 `false`。

---

## 示例

**示例 1**
```text
输入：
matrix = [
  [1, 4, 7, 11, 15],
  [2, 5, 8, 12, 19],
  [3, 6, 9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
],
target = 5

输出：true
```

---

## 题目本质

这题和普通二维数组查找不一样，因为它同时具有两个单调性：行有序且列有序。这并不意味着矩阵展开后是整体有序的，因此不能直接对整个矩阵进行一次二分查找。但我们可以利用“行列单调”的性质去排除大量元素。

---

## 错误直觉：为什么不能直接整体二分

有些矩阵题（如 LeetCode 74）可以将矩阵看作整体升序数组，因为上一行末尾小于下一行开头。但本题中，上一行末尾（如 7）可能大于下一行开头（如 2），展开后并非整体升序，所以不能直接用一次二分解决。

---

## 最优思路：从右上角开始搜索

这是本题最经典、最推荐的解法。

### 为什么选右上角

从右上角 `matrix[row][col]` 开始搜索具有非常好的性质：
* **左边的数都更小**
* **下面的数都更大**

比较 `target` 和当前值后，可以明确知道下一步的走向。

### 具体规则

1.  **`target == matrix[row][col]`**：找到目标，返回 `true`。
2.  **`target < matrix[row][col]`**：由于当前列从上到下递增，当前元素下方的所有元素都更大，不可能是答案。因此排除当前列，**左移**（`col--`）。
3.  **`target > matrix[row][col]`**：由于当前行从左到右递增，当前元素左边的所有元素都更小，不可能是答案。因此排除当前行，**下移**（`row++`）。

---

## 算法流程

1.  从右上角出发：`row = 0`, `col = n - 1`。
2.  当 `row` 没出界且 `col` 没出界时：
    * 若当前值等于目标值，返回 `true`。
    * 若目标值更小，左移。
    * 若目标值更大，下移。
3.  如果走出边界还没找到，返回 `false`。

---

## C 语言代码实现

```c
#include <stdbool.h>

bool searchMatrix(int** matrix, int matrixSize, int* matrixColSize, int target) {
    if (matrixSize == 0 || matrixColSize[0] == 0) {
        return false;
    }

    int row = 0;
    int col = matrixColSize[0] - 1;

    while (row < matrixSize && col >= 0) {
        if (target < matrix[row][col]) {
            col--;
        }
        else if (target > matrix[row][col]) {
            row++;
        }
        else {
            return true;
        }
    }

    return false;
}
```

---

## 结合示例理解过程

以 `target = 5` 为例：
1.  **右上角开始**：来到 `15`。$5 < 15$，左移。
2.  **来到 `11`**：$5 < 11$，左移。
3.  **来到 `7`**：$5 < 7$，左移。
4.  **来到 `4`**：$5 > 4$，下移。
5.  **来到 `5`**：$5 == 5$，找到目标，返回 `true`。

---

## 复杂度分析

* **时间复杂度**：每一路要么 `row++`，要么 `col--`，总共最多移动 $m + n$ 步，因此复杂度为 $$O(m + n)$$。
* **空间复杂度**：只用了常数个辅助变量，复杂度为 $$O(1)$$。

---

## 这题还有没有其他做法

* **逐行/逐列二分查找**：对每一行或每一列做二分，时间复杂度为 $O(m \log n)$ 或 $O(n \log m)$。
* **分治法**：从中间切分矩阵递归搜索，写法较复杂，面试中不如右上角搜索直观。

---

## 易错点总结

* **不要与 LC 74 混淆**：LC 74 可以整体二分，本题不行。
* **选择正确的起始点**：必须选择右上角或左下角，因为这两个位置具有明确的“双向单调性”。
* **处理空矩阵**：务必增加 `if (matrixSize == 0 || matrixColSize[0] == 0)` 的判断以避免越界。

---

## 面试答题模板

> 这个矩阵每行递增、每列递增，因此适合从右上角开始搜索。右上角元素左边都更小，下面都更大。
> 如果目标值小于当前值，说明当前列不可能包含答案，向左移动；
> 如果目标值大于当前值，说明当前行不可能包含答案，向下移动；
> 相等则返回 `true`。
> 这样每步排除一整行或一整列，时间复杂度为 $O(m + n)$，空间复杂度为 $O(1)$。

---

## 一句话总结

**搜索二维矩阵 II 的核心是：从右上角开始，利用双向单调性每次排除一整行或一整列。**
