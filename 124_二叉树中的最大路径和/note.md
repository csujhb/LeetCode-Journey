# 📚 算法学习笔记：二叉树中的最大路径和



##  题目核心定义

* **路径**：被定义为一条从树中任意节点出发，沿父节点-子节点连接，达到任意节点的序列。同一个节点在一条路径序列中至多出现一次。
* **路径和**：路径中各节点值的总和。
* **目标**：找出一个二叉树中路径和最大的路径，并返回该最大和。（注意：节点值可能为负数）。

---

##  解题思路：后序遍历 + 贪心剪枝

这道题无法自顶向下解决，必须采用**后序遍历（自底向上）**。每个节点需要向它的父节点汇报自己的情况，同时在内部挑战全局最大值。

### 核心拆解：两个关键动作

对于任意一个节点 `root`，我们需要做两件事：

1. **计算以当前节点为顶点的“内部最大路径和”并挑战全局变量**
   * 路径可以“左边连着右边”穿过当前节点，形成一个倒 V 型。
   * 内部路径和 = `root->val + 左子树的正向贡献 + 右子树的正向贡献`。
   * 用这个和去更新全局的 `maxSum`。

2. **向父节点返回“最大单边贡献值”**
   * 对于父节点来说，当前节点只能作为一条单向分支接入。
   * 单边贡献值 = `root->val + MAX(左子树正向贡献, 右子树正向贡献)`。
   * **贪心剪枝（精髓）**：如果子树算出来的单边贡献是负数，那我们干脆**不要它**（视为 0），因为加上负数只会拖累总和！

---

###  ⚠️ 避坑指南：C 语言宏定义陷阱（血泪教训）

在 C 语言中，我们经常使用宏来定义求最大值：
`#define MAX(a,b) ((a) > (b) ? (a) : (b))`

**【绝对禁止的写法】（会导致 TLE 超时）：**
```c
// 错误示范！将导致 O(N) 退化为 O(2^N) 的指数级复杂度
int left = MAX(0, dfs(root->left, maxSum)); 
```
* **原因**：宏会在预编译阶段直接展开为 `((0) > (dfs(...)) ? (0) : (dfs(...)))`。如果条件不满足，`dfs` 递归函数会被**执行两次**！在庞大的二叉树中，这会导致时间复杂度爆炸。

**【正确的写法】：**
```c
// 先用变量接收递归返回值，再参与宏计算
int left_val = dfs(root->left, maxSum);
int left = MAX(0, left_val); 
```

---

###  C 语言标准代码实现

```c
#include <limits.h>

// 安全的宏调用前提：传入的参数必须是单纯的变量，不能是包含副作用或耗时的函数！
#define MAX(a,b) ((a) > (b) ? (a) : (b))

// 递归函数：返回当前节点能提供的最大单边贡献值
int dfs(struct TreeNode* root, int* maxSum) {
    if (root == NULL) {
        return 0; // 空节点贡献为 0
    }

    // 1. 递归获取左右子树的原始贡献值（必须先存入变量避开宏陷阱）
    int left_val = dfs(root->left, maxSum);
    int right_val = dfs(root->right, maxSum);

    // 2. 贪心剪枝：如果贡献为负，直接舍弃（视为0）
    int left = MAX(0, left_val);
    int right = MAX(0, right_val);

    // 3. 挑战全局最大值：内部路径穿过当前节点
    int currentPathSum = root->val + left + right;
    *maxSum = MAX(*maxSum, currentPathSum);

    // 4. 向上汇报：返回当前节点作为单边路径的最大贡献
    return root->val + MAX(left, right);
}

int maxPathSum(struct TreeNode* root) {
    // 题目节点值可能全为负数，全局最大值初始必须设为最小整数
    int maxSum = INT_MIN; 
    dfs(root, &maxSum);
    return maxSum;
}
```

---

##  复杂度分析

* **时间复杂度：$O(N)$**。其中 $N$ 为二叉树节点的个数。每个节点在递归中只被遍历一次。
* **空间复杂度：$O(N)$**。空间复杂度主要取决于递归调用栈的深度。最坏情况下（树退化为单链表），树的高度为 $N$，需要 $O(N)$ 的栈空间；平均情况下（平衡树），栈空间为 $O(\log N)$。
