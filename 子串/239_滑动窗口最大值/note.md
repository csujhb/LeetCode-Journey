# 力扣 239. 滑动窗口最大值 学习笔记

---

## 题目描述

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组最左侧移动到最右侧。窗口每次只向右移动一位。

要求返回每次窗口移动后，窗口中的 **最大值**。

---

## 示例

**示例 1**

```text
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
```

**解释：**

* 窗口 `[1,3,-1]`，最大值是 `3`
* 窗口 `[3,-1,-3]`，最大值是 `3`
* 窗口 `[-1,-3,5]`，最大值是 `5`
* 窗口 `[-3,5,3]`，最大值是 `5`
* 窗口 `[5,3,6]`，最大值是 `6`
* 窗口 `[3,6,7]`，最大值是 `7`

---

## 题目本质

这题本质上是在问：

> 对于一个不断向右滑动的固定长度窗口，怎样快速维护窗口中的最大值？

**难点在于：**
* 窗口每次移动时，会删除一个旧元素。
* 会加入一个新元素。
* 需要快速知道当前窗口最大值。

如果每次都重新扫描窗口找最大值，复杂度会变得很高。

---

## 暴力解法

**思路**
每形成一个窗口，就遍历窗口中的 `k` 个元素，找最大值。

**代码（C语言）**
```c
int* maxSlidingWindow(int* nums, int numsSize, int k, int* returnSize) {
    *returnSize = numsSize - k + 1;
    int* ans = (int*)malloc(sizeof(int) * (*returnSize));

    for (int i = 0; i <= numsSize - k; i++) {
        int maxVal = nums[i];
        for (int j = i; j < i + k; j++) {
            if (nums[j] > maxVal) {
                maxVal = nums[j];
            }
        }
        ans[i] = maxVal;
    }
    return ans;
}
```

**复杂度分析**
* 时间复杂度：$O((n-k+1) \times k)$，最坏可写作 $O(nk)$。
* 空间复杂度：$O(1)$（不算返回数组）。

**缺点**
当 `numsSize` 和 `k` 都很大时，这种做法会超时。

---

## 最优解：单调队列

这题的标准最优解是使用 **单调队列（Monotonic Queue）**。


---

## 为什么普通队列不行

如果我们用普通队列保存窗口元素：
* 能知道谁先进谁后出。
* 但不能快速知道最大值。

如果再配一个“当前最大值”变量，当最大值被移出窗口时，就没法快速得到新的最大值。因此需要一种结构，既能维护窗口顺序，又能快速得到最大值，这就是单调队列。

---

## 单调队列的核心思想

维护一个 **单调递减队列**，队列中存的是 **数组下标**，而不是值。

并保证：
```text
nums[deque[front]] >= nums[deque[front+1]] >= ... >= nums[deque[rear]]
```

也就是说：
* 队首始终是当前窗口中最大的元素下标。
* 队列里保存的是“可能成为窗口最大值的候选元素”。

---

## 为什么要存下标而不是值

因为我们需要判断某个元素是否已经滑出窗口。

例如当前窗口右端点是 `i`，窗口范围是 `[i-k+1, i]`。如果队首下标满足 `deque[front] <= i - k`，说明这个元素已经不在窗口里了，需要移除。如果只存值，就无法判断它是否过期。

---

## 如何维护单调递减队列

当遍历到新元素 `nums[i]` 时，做两件事：


**移除队首中过期元素**
如果队首元素下标已经不在窗口范围内，就从队首弹出。

**从队尾删除所有比当前元素小（或小于等于）的元素**
因为这些元素以后不可能成为最大值了。
原因：
* 它们比当前元素小。
* 它们还比当前元素更早进入窗口，会更早过期。
* 在它们过期之前，当前元素已经把它们“压住了”。

---

## 算法流程

遍历数组下标 `i`：
1. 移除队首中过期的下标。
2. 从队尾删除所有值小于等于当前值的下标。
3. 把当前下标加入队列。
4. 当窗口大小达到 `k` 时，队首对应的值就是当前窗口最大值。

---

## C语言标准实现

```c
#include <stdlib.h>

int* maxSlidingWindow(int* nums, int numsSize, int k, int* returnSize) {
    *returnSize = numsSize - k + 1;
    int* result = (int*)malloc(sizeof(int) * (*returnSize));
    int* deque = (int*)malloc(sizeof(int) * numsSize);  // 存下标

    int front = 0, rear = -1;

    for (int i = 0; i < numsSize; i++) {
        // 1. 移除队首过期元素
        while (front <= rear && deque[front] <= i - k) {
            front++;
        }

        // 2. 保持队列单调递减
        while (front <= rear && nums[deque[rear]] <= nums[i]) {
            rear--;
        }

        // 3. 当前元素下标入队
        deque[++rear] = i;

        // 4. 窗口形成后，记录最大值
        if (i >= k - 1) {
            result[i - k + 1] = nums[deque[front]];
        }
    }

    free(deque);
    return result;
}
```

---

## 你的代码优化建议

**过期元素建议用 `while`**
你原来写的是 `if`。虽然本题每次最多过期一个，但 `while` 更稳妥、通用。

**队尾比较可以写成 `<=`**
你原来写的是 `<`。使用 `<=` 会删除旧的相等元素，保留更新的下标，使队列更简洁。

**空间分配**
你现在分配了 `numsSize` 大小的空间给 `deque`。从理论上说，队列有效元素最多约为 `k` 个，这种实现虽然空间为 $O(n)$，但在 LeetCode 中是完全可接受的。

---

## 为什么时间复杂度是 $O(n)$

虽然有嵌套的 `while`，但：
* 每个元素最多入队一次。
* 每个元素最多从队尾弹出一次。
* 每个元素最多从队首弹出一次。
总操作次数与 $n$ 成正比，因此总时间复杂度为 $O(n)$。

---

## 其他解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优缺点 |
| :--- | :--- | :--- | :--- |
| **单调队列** | $O(n)$ | $O(k)$ | 最优解，最高效 |
| **堆 (优先队列)** | $O(n \log k)$ | $O(k)$ | 复杂度较高 |
| **分块预处理** | $O(n)$ | $O(n)$ | 线性时间，但实现较复杂 |
| **暴力法** | $O(nk)$ | $O(1)$ | 直接超时 |

---

## 易错点总结

* **队列存下标**：因为需要通过下标判断元素是否过期。
* **顺序**：先删过期元素，再加入当前元素。
* **单调性**：求最大值用 **单调递减队列**；求最小值用 **单调递增队列**。
* **窗口形成条件**：只有当 `i >= k - 1` 时才开始记录结果。
