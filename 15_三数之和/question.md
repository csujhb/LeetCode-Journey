# 15. 三数之和

## 题目描述

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

## 示例

**示例 1：**

> **输入：** nums = [-1,0,1,2,-1,-4]
>
> **输出：** [[-1,-1,2],[-1,0,1]]
>
> **解释：** > nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
>
> nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
>
> nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
>
> 不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
>
> 注意，输出的顺序和三元组的顺序并不重要。

**示例 2：**

> **输入：** nums = [0,1,1]
>
> **输出：** []
>
> **解释：** 唯一可能的三元组和不为 0 。

**示例 3：**

> **输入：** nums = [0,0,0]
>
> **输出：** [[0,0,0]]
>
> **解释：** 唯一可能的三元组和为 0 。

## 提示

- $3 \le nums.length \le 3000$
- $-10^5 \le nums[i] \le 10^5$

## 进阶

你能否设计一个时间复杂度为 $O(n^2)$ 的算法解决此问题？尝试使用 **排序 + 双指针** 的策略，并思考在移动指针时如何跳过相同的元素以达到去重的目的。
