# 力扣 73. 矩阵置零（C 语言）学习笔记

------

## 题目描述

给定一个 `m x n` 的矩阵 `matrix`，如果一个元素为 `0`，则将其所在行和所在列的所有元素都设为 `0`。

要求：

- 尽量 **原地** 修改矩阵
- 进阶要求：尝试使用 **常量空间** 完成

------

## 题意理解

### 示例 1

输入：

Plaintext

```
[
  [1,1,1],
  [1,0,1],
  [1,1,1]
]
```

输出：

Plaintext

```
[
  [1,0,1],
  [0,0,0],
  [1,0,1]
]
```

因为中间那个 `0` 位于第 1 行、第 1 列，所以整行整列都要变为 `0`。

------

### 示例 2

输入：

Plaintext

```
[
  [0,1,2,0],
  [3,4,5,2],
  [1,3,1,5]
]
```

输出：

Plaintext

```
[
  [0,0,0,0],
  [0,4,5,0],
  [0,3,1,0]
]
```

------

## 容易犯的错误

这法子最容易犯的错误是：一边遍历矩阵，一边直接把所在行列清零。这样会导致“新产生的 0”继续扩散，错误地影响后续判断。

### 错误示例

如果扫描到 `matrix[1][1] == 0` 后立刻把第 1 行和第 1 列清零，接着继续扫描时会看到新出现的 0，从而错误地把更多行列清零。因此必须分成两步：

1. 先记录哪些行、哪些列需要清零。
2. 再统一修改。

------

## 你的原始思路：使用行标记数组 + 列标记数组

这是一种很常见、也很容易想到的方法。

### 思路

- 用一个数组 `row[]` 记录哪些行要清零。
- 用一个数组 `col[]` 记录哪些列要清零。
- 第一遍扫描矩阵，遇到 `0` 就标记对应行和列。
- 第二遍根据标记清零。

------

## 你原代码中的一个问题

你之前代码中有这段：

C

```
if (matrix[i][j] == 0) {
    row[i] = 1;
    col[j] = 1;
    break;
}
```

这里使用 `break` 会导致某一行一旦遇到第一个 `0` 就提前结束扫描，该行后面如果还有别的 `0`，对应的列就不会被标记，导致标记不完整。

------

## O(m+n) 空间解法

### 思路

定义：

- `row[i] = 1` 表示第 `i` 行需要清零。
- `col[j] = 1` 表示第 `j` 列需要清零。

步骤：

1. 第一遍遍历矩阵，遇到 `matrix[i][j] == 0` 时设置 `row[i] = 1` 和 `col[j] = 1`。
2. 第二遍遍历矩阵，如果 `row[i] == 1 || col[j] == 1`，则把 `matrix[i][j] = 0`。

### C 语言代码

C

```
#include <stdlib.h>

void setZeroes(int** matrix, int matrixSize, int* matrixColSize) {
    int m = matrixSize;
    int n = matrixColSize[0];

    int* row = (int*)calloc(m, sizeof(int));
    int* col = (int*)calloc(n, sizeof(int));

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (matrix[i][j] == 0) {
                row[i] = 1;
                col[j] = 1;
            }
        }
    }

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (row[i] == 1 || col[j] == 1) {
                matrix[i][j] = 0;
            }
        }
    }

    free(row);
    free(col);
}
```

### 复杂度分析

- 时间复杂度：`O(m*n)`
- 空间复杂度：`O(m+n)`

------

## 进阶：常量空间 O(1) 解法

### 核心思想

直接用矩阵的第一行和第一列来充当标记数组：

- 用 `matrix[i][0]` 标记第 `i` 行是否需要清零。
- 用 `matrix[0][j]` 标记第 `j` 列是否需要清零。

### 难点

第一行和第一列本身也可能原来就有 `0`。因此需要两个额外变量记录：

- `firstRowZero`：第一行原本是否有 `0`。
- `firstColZero`：第一列原本是否有 `0`。

------

## 常量空间解法步骤

1. **检查第一行是否有 0**：如果原本有 `0`，最后整行都要清零。
2. **检查第一列是否有 0**：如果原本有 `0`，最后整列都要清零。
3. **用第一行和第一列做标记**：从 `matrix[1][1]` 开始扫描，若 `matrix[i][j] == 0`，则令 `matrix[i][0] = 0` 且 `matrix[0][j] = 0`。
4. **根据标记清零中间区域**：对所有 `i >= 1` 且 `j >= 1` 的位置，若 `matrix[i][0] == 0 || matrix[0][j] == 0`，则 `matrix[i][j] = 0`。
5. **最后处理第一行和第一列**：因为它们既是数据区又是标记区，必须最后单独处理。

------

## 常量空间解法代码（C 语言）

C

```
void setZeroes(int** matrix, int matrixSize, int* matrixColSize) {
    int m = matrixSize;
    int n = matrixColSize[0];

    int firstRowZero = 0;
    int firstColZero = 0;

    for (int j = 0; j < n; j++) {
        if (matrix[0][j] == 0) {
            firstRowZero = 1;
            break;
        }
    }

    for (int i = 0; i < m; i++) {
        if (matrix[i][0] == 0) {
            firstColZero = 1;
            break;
        }
    }

    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            if (matrix[i][j] == 0) {
                matrix[i][0] = 0;
                matrix[0][j] = 0;
            }
        }
    }

    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            if (matrix[i][0] == 0 || matrix[0][j] == 0) {
                matrix[i][j] = 0;
            }
        }
    }

    if (firstRowZero) {
        for (int j = 0; j < n; j++) matrix[0][j] = 0;
    }

    if (firstColZero) {
        for (int i = 0; i < m; i++) matrix[i][0] = 0;
    }
}
```

------

## 常量空间解法详细举例

以矩阵示例 1 为例：

1. **检查首行首列**：`firstRowZero = 0`, `firstColZero = 0`。
2. **打标记**：扫描到 `matrix[1][1] == 0`，设置 `matrix[1][0] = 0` 和 `matrix[0][1] = 0`。
3. **处理内部**：`matrix[1][2]` 因为 `matrix[1][0]` 为 0 而变 0；`matrix[2][1]` 因为 `matrix[0][1]` 为 0 而变 0。
4. **处理首行首列**：由于变量均为 0，首行首列保持现状。

------

## 为什么这种方法是 O(1) 空间

因为没有额外申请与矩阵规模相关的数组，只用了两个固定大小的变量 `firstRowZero` 和 `firstColZero`。无论矩阵多大，空间占用都是固定的。

------

## 两种方法对比

| **方法**           | **思路**                  | **时间复杂度** | **空间复杂度** | **难度** |
| ------------------ | ------------------------- | -------------- | -------------- | -------- |
| 行列标记数组       | 额外开 `row[]` 和 `col[]` | `O(m*n)`       | `O(m+n)`       | 简单     |
| 第一行第一列作标记 | 借用原矩阵存标记          | `O(m*n)`       | `O(1)`         | 较难     |

------

## 易错点总结

- **不能一边遍历一边清零**：否则新产生的 0 会扩散。
- **不能提前 `break`**：每一行中的每个 0 都必须标记其对应的列。
- **常量空间法需先保存首行首列状态**：防止标记过程覆盖原始信息。
- **内部区域要从索引 `1` 开始遍历**：避免破坏首行首列的标记作用。
- **首行首列要最后处理**：它们既是数据又是标记。

------

## 面试时可以怎么说

可以分为两个层次回答：

1. **普通做法**：使用两个额外数组记录需要清零的行和列，空间复杂度为 `O(m+n)`。
2. **进阶做法**：利用矩阵的第一行和第一列作为标记位，再用两个变量保存它们原本的状态，从而将空间复杂度降至 `O(1)`。

------

## 推荐记忆模板

> 第一行标列，第一列标行；先存首行首列状态，中间打标记，最后统一清零。
