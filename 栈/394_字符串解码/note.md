# 📚 算法学习笔记：字符串解码 (LeetCode 394)

##  题目核心定义

- **目标**：给定一个经过编码的字符串，返回它解码后的字符串。
- **编码规则**：`k[encoded_string]`，表示内部的 `encoded_string` 正好重复 `k` 次。
- **难点**：括号是可以**嵌套**的（如 `3[a2[c]]`），必须由内而外依次解开。

------

##  核心解题思想：递归下降 (DFS)

面对具有嵌套性质的问题（如括号匹配、树的遍历、表达式求值），**栈 (Stack)** 和 **递归 (Recursion)** 是硬币的正反面。

在 C 语言中，由于没有现成的高级容器，手写一个能同时存储“数字”和“动态字符串”的栈非常繁琐。因此，**利用系统调用栈的“递归法”是最优雅的解法**。

### 递归的状态转移逻辑：

遍历字符串，根据当前字符分四种情况处理：

1. **数字 (`0-9`)**：可能是多位数，持续累加计算重复次数 `k`。
2. **左括号 (`[`)**：**进入下一层嵌套！** 调用递归函数自身去处理括号里面的内容。等递归返回时，把拿到的子串重复 `k` 次拼接到当前结果中。
3. **右括号 (`]`)**：**退出当前嵌套！** 结束当前层的遍历，把拼好的结果返回给上一层。
4. **普通字母**：直接拼接到当前结果的尾部。

------

## ⚠️ 核心技巧与避坑指南 (C 语言专属)

这道题在 C 语言里的工程难点，远远大于算法逻辑本身。必须跨过以下三个大坑：

### 坑 1：必须使用“全局共享”的索引指针

- **错误做法**：在递归函数里按值传递索引（比如 `dfs(char* s, int i)`），当内层递归处理完 `2[c]` 返回后，外层依然不知道内层处理到了哪里，会导致重复解析。
- **正确做法**：传递索引的指针 `dfs(char* s, int* i)`。这样，哪怕在最深层的递归里移动了指针，每一层也都能同步感知到字符串已经被解析到哪个位置了。

### 坑 2：多位数字的陷阱

- 测试用例不仅有 `3[a]`，还有可能是 `100[leetcode]`。
- 读取数字时不能直接 `k = c - '0'`，必须使用累加法：`k = k * 10 + (c - '0')`。

### 坑 3：动态扩容与内存泄漏

- 解码后的字符串长度是完全未知的（比如 `100[a100[b]]` 会炸开成非常长的字符串）。
- 必须使用 `malloc` 赋初值，并用 `realloc` 动态扩容。
- **极易扣分点**：拿到内层递归返回的子串并拼接完之后，**必须立刻 `free` 掉子串**，否则会产生严重的内存泄漏。

##  C 语言标准代码模板

C

```
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

// 核心递归函数，传入字符串和全局共享的索引指针 i
char* dfs(char* s, int* i) {
    int capacity = 50; 
    char* res = (char*)malloc(capacity * sizeof(char));
    res[0] = '\0';
    int len = 0; 
    int k = 0;   // 记录重复次数

    while (s[*i] != '\0') {
        char c = s[*i];

        if (isdigit(c)) {
            // 处理多位数字
            k = k * 10 + (c - '0');
            (*i)++;
        } 
        else if (c == '[') {
            // 核心转折：跳过 '['，开启新一层递归！
            (*i)++; 
            char* subStr = dfs(s, i); 
            (*i)++; // 递归回来时 *i 指向 ']'，跳过它

            int subLen = strlen(subStr);
            // 动态扩容
            while (len + k * subLen + 1 > capacity) {
                capacity *= 2;
                res = (char*)realloc(res, capacity * sizeof(char));
            }
            
            // 将子串重复 k 次追加到 res 尾部
            for (int j = 0; j < k; j++) {
                strcpy(res + len, subStr);
                len += subLen;
            }
            
            k = 0; // 清零数字，准备下一段
            free(subStr); // 【极其重要】释放内层内存！
        } 
        else if (c == ']') {
            // 遇到右括号，当前层任务完成，立刻跳出循环返回
            break;
        } 
        else {
            // 普通字符，直接追加
            if (len + 2 > capacity) {
                capacity *= 2;
                res = (char*)realloc(res, capacity * sizeof(char));
            }
            res[len++] = c;
            res[len] = '\0';
            (*i)++;
        }
    }
    return res;
}

char* decodeString(char* s) {
    int i = 0;
    return dfs(s, &i);
}
```

------

##  复杂度分析

- **时间复杂度**：$O(L)$。其中 $L$ 是**解码后最终字符串的长度**。虽然看起来是 $O(N)$ 遍历原字符串，但因为我们要频繁进行字符串拼接操作，实际上要把最终结果的每一个字符都物理写入内存一遍。
- **空间复杂度**：$O(L)$。主要消耗在于存储最终结果的数组 `res`，以及递归调用的栈深度（取决于括号嵌套的层数，最坏不超过原字符串长度 $N$）。
