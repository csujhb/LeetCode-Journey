# 📚 算法学习笔记：有效的括号 (LeetCode 20)

##  题目核心定义

* **目标**：给定一个只包括 `'('`，`')'`，`'{'`，`'}'`，`'['`，`']'` 的字符串 `s`，判断字符串是否有效。
* **有效条件**：
  1. 左括号必须用相同类型的右括号闭合。
  2. 左括号必须以正确的顺序闭合。
  3. 每个右括号都有一个对应的相同类型的左括号。

---

##  核心解题思想：栈 (Stack)



解决括号匹配问题的唯一真神是**栈**。
* **为什么用栈？** 括号的匹配规则天然具有 **“后进先出”（LIFO）** 的特性。最后被写下的左括号，必须最先被对应的右括号闭合（例如 `[{()}]`，最内层的 `()` 必须最先消解）。
* **核心动作**：
  * **左括号**：代表“期待”，无脑**入栈**。
  * **右括号**：代表“还债”，必须与栈顶的左括号配对，配对成功则**出栈**（消解）。

---

##  算法状态机推演

遍历字符串的每一个字符，只有以下三种状态：

1. **遇到左括号** `(`, `[`, `{`：
   * 压入栈中，指针后移。
2. **遇到右括号** `)`, `]`, `}`：
   * **情况 A（栈为空）**：没有左括号能和它匹配（如 `"]"`），直接判定非法。
   * **情况 B（栈顶不匹配）**：类型不对（如栈顶是 `(`，当前是 `]`），直接判定非法。
   * **情况 C（栈顶匹配）**：一对括号成功消解，栈顶元素出栈，指针后移。
3. **遍历结束**：
   * **栈为空**：所有括号完美配对，返回 `true`。
   * **栈不为空**：还有左括号在“苦苦等待”右括号（如 `"((("`），返回 `false`。

---

##  ⚠️ 避坑指南（关键易错点）

基于手写 C 语言数组模拟栈时的常见错误：

* **漏洞 1：忘记移动指针 (`s++`)**
  * 在复杂的 `if-else` 或 `switch` 匹配成功后，极易忘记让字符串指针前进，导致死循环或重复匹配。
  * **对策**：将 `s++` 放在 `while` 循环的底部，作为每一轮迭代必做的收尾动作。
* **漏洞 2：空栈强行入右括号**
  * 当 `top == -1` 时遇到右括号，如果不加拦截直接压栈，会破坏栈内只能存放左括号的规矩。
  * **对策**：遇到右括号时，第一件事就是检查 `if (top == -1) return false;`。
* **漏洞 3：结尾默认返回 `true`**
  * 循环跑完只代表“没有遇到冲突的右括号”，不代表“所有左括号都被闭合了”。
  * **对策**：结尾必须写成 `return top == -1;`，用栈是否清空来做最终裁决。

##  C 语言标准代码实现

```c
#include <stdbool.h>

#define STACK_SIZE 10000

bool isValid(char* s) {
    char stack[STACK_SIZE];
    int top = -1; // -1 表示空栈

    while (*s != '\0') {
        // 1. 遇到左括号，无条件入栈
        if (*s == '(' || *s == '{' || *s == '[') {
            stack[++top] = *s;
        } 
        // 2. 遇到右括号
        else {
            // 防御：如果遇到右括号时栈为空，说明多出了右括号
            if (top == -1) return false; 

            char cur = stack[top]; // 取栈顶元素准备匹配
            switch (*s) {
                case ')':
                    if (cur != '(') return false;
                    top--; // 匹配成功，出栈
                    break;
                case '}':
                    if (cur != '{') return false;
                    top--;
                    break;
                case ']':
                    if (cur != '[') return false;
                    top--;
                    break;
            }
        }
        
        // 3. 指针后移，处理下一个字符
        s++; 
    }

    // 4. 遍历完毕，栈空则说明全部合法
    return top == -1; 
}
```

##  复杂度分析

* **时间复杂度：$O(N)$**。其中 $N$ 为字符串长度，只需遍历一遍字符串，每次栈操作耗时为 $O(1)$。
* **空间复杂度：$O(N)$**。最坏情况下（全部为左括号），我们需要将所有字符压入栈中。
