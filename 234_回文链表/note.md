# 力扣 234. 回文链表 学习笔记

---

## 题目描述

给你一个单链表的头节点 `head`，判断该链表是否为回文链表。

如果是，返回 `true`；否则返回 `false`。

---

## 示例

### 示例 1

```text
输入：head = [1,2,2,1]
输出：true
```

---

### 示例 2

```text
输入：head = [1,2]
输出：false
```

---

## 题目本质

这题本质上是在问：如何判断一个单链表从前往后读和从后往前读是否相同？

对于数组来说，这件事很简单：直接双指针从两端往中间比较。

但链表没有随机访问能力，不能像数组那样直接拿到“倒数第几个元素”。
所以难点在于：**怎么在链表上模拟“两端比较”？**

---

## 常见做法概述

### 转成数组
* **思路**：遍历链表，把所有值存进数组，然后用双指针判断数组是否回文。
* **复杂度**：时间复杂度 $O(n)$，空间复杂度 $O(n)$。
* **优缺点**：最简单，但由于不满足题目进阶要求的 $O(1)$ 额外空间，通常仅作为保底方案。

### 栈
* **思路**：遍历链表把值压入栈，再遍历一次链表并与栈顶元素比对；或者只压入前半段。
* **复杂度**：时间复杂度 $O(n)$，空间复杂度 $O(n)$。
* **缺点**：仍然使用了额外空间。

### 递归
* **思路**：利用递归先走到链表尾部，再在回溯阶段与头部同步比较。
* **复杂度**：时间复杂度 $O(n)$，空间复杂度 $O(n)$（递归栈）。
* **缺点**：链表过长时可能导致栈溢出。

### 快慢指针 + 反转后半段链表
这是这题最经典、最优的做法。
* **复杂度**：时间 $O(n)$，空间 $O(1)$。
* **特点**：满足进阶要求，面试中最推荐的标准解法。

---

## 最优解核心思路

该做法可以分成三步：
1.  **快慢指针**找到链表中点。
2.  **反转**后半段链表。
3.  从两端向中间**逐个比较**。

---

## 快慢指针找中点


定义两个指针：
* `slow` 每次走一步。
* `fast` 每次走两步。
当 `fast` 走到链表末尾时，`slow` 就到达中点附近。

### 偶数长度情况
例如 `1 -> 2 -> 2 -> 1`，最终 `slow` 指向后半段起点，`fast == NULL`。

### 奇数长度情况
例如 `1 -> 2 -> 3 -> 2 -> 1`，最终 `slow` 指向中间节点 `3`，`fast != NULL`。
此时中间节点不影响回文性，通常会执行 `slow = slow->next` 跳过它。

---

## 反转后半段链表


反转后半段后，就可以把“前半段从左往右”和“后半段从右往左”变成两个都能顺着 `next` 指针前进的链表。

例如原链表 `1 -> 2 -> 2 -> 1`，后半段 `2 -> 1` 反转后变为 `1 -> 2`。
现在只需拿前半段 `1 -> 2` 与反转后的后半段 `1 -> 2` 逐个比较即可。

---

## 逐节点比较

定义两个指针：
* `first` 指向头结点。
* `second` 指向反转后的后半段头结点。

同时往后走，如果某一对值不同，返回 `false`；如果全部相同，返回 `true`。

---

## 标准 C 代码实现

```c
#include <stdbool.h>
#include <stdlib.h>

struct ListNode {
    int val;
    struct ListNode *next;
};

// 辅助函数：反转链表
struct ListNode* reverseList(struct ListNode* head) {
    struct ListNode* prev = NULL;
    struct ListNode* cur = head;
    while (cur != NULL) {
        struct ListNode* next = cur->next;
        cur->next = prev;
        prev = cur;
        cur = next;
    }
    return prev;
}

bool isPalindrome(struct ListNode* head) {
    if (head == NULL || head->next == NULL) return true;

    struct ListNode* slow = head;
    struct ListNode* fast = head;

    // 1. 找中点
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
    }

    // 2. 奇数长度时跳过中间节点
    if (fast != NULL) {
        slow = slow->next;
    }

    // 3. 反转后半段
    struct ListNode* second = reverseList(slow);
    struct ListNode* p1 = head;
    struct ListNode* p2 = second;

    // 4. 比较两半
    bool result = true;
    while (p2 != NULL) {
        if (p1->val != p2->val) {
            result = false;
            break;
        }
        p1 = p1->next;
        p2 = p2->next;
    }

    // 5. 恢复链表（养成良好习惯，还原输入结构）
    reverseList(second);

    return result;
}
```

---

## 你当前代码的思路说明

你写的代码（找中点 -> 从 `slow` 开始反转 -> 比较）也是正确的。
由于你没有区分奇偶逻辑，在奇数长度链表（如 `1->2->3->2->1`）中，`slow` 指向 `3`，反转后变为 `1->2->3`。与头节点开始的路径比较，依然能得到正确结果。

---

## 你的写法和标准写法的区别

### 你的写法特点
* 不区分奇偶，直接从 `slow` 开始反转。
* 在奇数情况下中间节点会被包含在反转部分中，逻辑依然自洽。

### 标准写法特点
* 显式区分奇偶，奇数时跳过中间节点。
* 语义更直观，明确了“前半段”与“后半段”的对比。

---

## 你的代码的优点与不足

* **优点**：复杂度已达最优 $O(n)$ 时间和 $O(1)$ 空间；封装独立，实现简单。
* **不足**：
    1.  **破坏原链表结构**：比较完后没有将链表恢复。虽然 LeetCode 不强制要求，但在工程和面试中，恢复链表是加分项。
    2.  **语义不够显式**：虽然逻辑正确，但“跳过中点”的写法在面试交流中更易于解释。

---

## 为什么这题已经没有更优解法？

你的做法已经达到了渐进复杂度的下界：
* **时间下界**：必须至少遍历一遍链表，故无法优于 $O(n)$。
* **空间下界**：只用了常数个指针变量，故额外空间已是 $O(1)$。

---

## 复杂度分析

* **时间复杂度**：$O(n)$。找中点、反转、比较各需一次或半次遍历，总和仍为线性。
* **空间复杂度**：$O(1)$。仅使用了若干指针变量。

---

## 易错点总结

* **奇偶长度处理**：是否跳过中点要根据自己的反转逻辑决定。
* **反转范围**：注意是反转“后半段”而不是整个链表。
* **比较终止条件**：通常以 `second != NULL` 为准，因为后半段长度总是小于或等于前半段。
* **边界情况**：空链表和单节点链表默认是回文的。
* **副作用**：如果面试官关心原数据完整性，记得最后执行一次反转还原。

---

## 面试答题模板

> “判断单链表是否为回文，最优做法是利用快慢指针寻找中点。
> 找到中点后，原地反转后半段链表，随后从头节点和反转后的后半段起点同时向后比较。
> 如果链表长度为奇数，可以跳过中间节点，因为它不影响回文性。
> 比较完成后，为了消除副作用，建议将后半段再反转回来还原链表结构。
> 该算法的时间复杂度为 $O(n)$，额外空间复杂度为 $O(1)$。”

---

## 一句话总结

**回文链表最优解 = 快慢指针找中点 + 原地反转后半链表 + 两端同步比较。**
