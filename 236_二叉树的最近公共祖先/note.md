# 📚 算法学习笔记：二叉树的最近公共祖先 (LCA)

##  题目核心定义

**最近公共祖先 (Lowest Common Ancestor)**：
对于有根树的两个节点 `p`、`q`，最近公共祖先表示为一个节点 `x`，满足 `x` 是 `p`、`q` 的祖先且 `x` 的深度尽可能大（一个节点也可以是它自己的祖先）。

---

## 解题思路：后序遍历（递归）

由于我们需要**自底向上**查找，而二叉树的递归天然具有“回溯”特性，因此使用**后序遍历**（左-右-根）是最优选择。

### 递归函数的意义：

`lowestCommonAncestor(root, p, q)` 函数的返回值有三种含义：
1. 如果以 `root` 为根的子树中包含 `p` 和 `q`，返回它们的**最近公共祖先**。
2. 如果只包含 `p`，返回 `p`；如果只包含 `q`，返回 `q`。
3. 如果都不包含，返回 `NULL`。

### 状态转移逻辑（核心）：

1. **终止条件**：当前节点为空，或者当前节点就是 `p` 或 `q`，直接返回当前节点。
2. **递归搜索**：去左子树找，去右子树找，分别得到返回值 `left` 和 `right`。
3. **结果判断**：
   - **左右开花 (`left && right`)**：说明 `p` 和 `q` 分别在当前节点的两侧，当前节点就是 LCA。
   - **单侧结果 (`left || right`)**：说明 `p` 和 `q` 都在同一侧，或者只找到了其中一个。返回那个非空的节点。
   - **空手而归 (`!left && !right`)**：返回 `NULL`。

---

##  C 语言代码实现

```c
struct TreeNode* lowestCommonAncestor(struct TreeNode* root, struct TreeNode* p, struct TreeNode* q) {
    // 1. 边界情况与终止条件
    if (root == NULL || root == p || root == q) {
        return root;
    }

    // 2. 递归查找左右子树 (后序遍历：下层信息向上层传递)
    struct TreeNode* left = lowestCommonAncestor(root->left, p, q);
    struct TreeNode* right = lowestCommonAncestor(root->right, p, q);

    // 3. 合并结果
    // 情况 A: 左右子树各发现一个目标节点，root 就是 LCA
    if (left != NULL && right != NULL) {
        return root;
    }
    
    // 情况 B: 只有左子树有发现（left包含LCA或p或q），返回 left
    if (left != NULL) {
        return left;
    }
    
    // 情况 C: 只有右子树有发现，返回 right
    return right;
}
```

---

## 关键点深度解析

* **为什么不需要专门找 p 再找 q？**
    递归过程中，如果遇到 `p` 我们就返回了。如果 `q` 在 `p` 的子树里，根据定义，`p` 就是它们的 LCA，直接返回 `p` 是正确的。
* **后序遍历的作用**：
    它保证了我们处理当前节点时，已经拿到了左右子树的“情报”。这种**自底向上**的消息传递是解决 LCA 问题的灵魂。

---

## 复杂度分析

* **时间复杂度：$O(N)$**
    其中 $N$ 是节点个数。最坏情况下我们需要遍历整棵二叉树。
* **空间复杂度：$O(N)$**
    主要消耗在递归调用的栈空间上。如果树退化为链表，深度为 $N$；如果是平衡二叉树，则为 $O(\log N)$。

## 衍生对比：普通二叉树 vs 二叉搜索树 (BST)

| 特性 | 普通二叉树 (236题) | 二叉搜索树 (235题) |
| :--- | :--- | :--- |
| **查找方式** | 必须遍历左右子树（DFS） | 根据数值大小判断方向 |
| **逻辑核心** | 左右子树返回值是否为空 | 节点值是否在 `[p, q]` 区间内 |
| **空间优化** | 递归必占 $O(H)$ 栈空间 | 可通过循环优化到 $O(1)$ |
