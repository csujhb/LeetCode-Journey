# 寻找重复数 (LeetCode 287) 核心学习笔记

---

## 快慢指针法（Floyd 判圈算法 / 龟兔赛跑）

**思路剖析：**
这是本题的绝对最优解，完美契合时间复杂度 $O(n)$ 且空间复杂度 $O(1)$ 并保留原数组的要求。将数组视为一个隐式链表，数组的下标作为“节点”，数组元素的值作为“指向下一个节点的指针”（即 `next = nums[i]`）。因为存在重复数字，必然存在多个下标指向同一个值，从而导致隐式链表中出现环。寻找重复数的问题就巧妙转化为“寻找链表环的入口”。
设置快慢指针，慢指针每次走一步，快指针每次走两步。两者相遇后，将快指针重置回起点，然后两指针同速前进，再次相遇的点即为环的入口，也就是重复的数字。

**C语言实现：**
```c
int findDuplicate(int* nums, int numsSize) {
    int slow = nums[0];
    int fast = nums[nums[0]];
    
    // 寻找环内相遇点
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[nums[fast]];
    }
    
    // 快指针回到起点，寻找环入口
    fast = 0;
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[fast];
    }
    
    return slow;
}
```

## 二分查找法

**思路剖析：**
这里的二分不是针对数组下标，而是针对数字的“值域”区间 `[1, n]`。利用抽屉原理，每次取值域的中位数 `mid`，遍历整个数组并统计小于等于 `mid` 的元素个数。如果统计个数严格大于 `mid`，说明重复数字一定落在左半区间 `[1, mid]`；否则，重复数字一定在右半区间 `[mid + 1, n]`。不断缩小值域区间，直到左右边界重合即可。此方法同样不修改原数组。

**C语言实现：**
```c
int findDuplicate(int* nums, int numsSize) {
    int left = 1, right = numsSize - 1;
    int ans = -1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        int count = 0;
        
        // 统计数组中小于等于 mid 的元素个数
        for (int i = 0; i < numsSize; i++) {
            if (nums[i] <= mid) {
                count++;
            }
        }
        
        if (count > mid) {
            ans = mid;
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }
    
    return ans;
}
```

## 二进制按位还原法

**思路剖析：**
这是一种通过位运算进行数字还原的巧妙思路。由于数组元素都在 `1` 到 `n` 之间，我们可以逐个考察整型数字的每一个二进制位（共计32位）。对于任意比特位，分别统计“原数组中所有元素在这一位上是 `1` 的数量”以及“标准序列 `1` 到 `n` 在这一位上是 `1` 的数量”。如果前者的数量严格大于后者，说明那个多出来的重复数字在该位上必定是 `1`。遍历所有比特位并把为 `1` 的位拼接起来，即可还原出重复数字。

**C语言实现：**
```c
int findDuplicate(int* nums, int numsSize) {
    int n = numsSize - 1;
    int ans = 0;
    
    // 确定最高位，优化外层循环次数
    int bit_max = 31;
    while (!((n >> bit_max) & 1)) {
        bit_max--;
    }
    
    for (int bit = 0; bit <= bit_max; bit++) {
        int x = 0, y = 0;
        for (int i = 0; i < numsSize; ++i) {
            // 统计原数组中该位为 1 的个数
            if (nums[i] & (1 << bit)) {
                x++;
            }
            // 统计 1 到 n 中该位为 1 的个数
            if (i >= 1 && (i & (1 << bit))) {
                y++;
            }
        }
        
        // 如果原数组该位 1 的个数偏多，说明重复数字该位是 1
        if (x > y) {
            ans |= (1 << bit);
        }
    }
    
    return ans;
}
```

## 原地哈希法（符号标记）

**思路剖析：**
此方法适用于**允许修改原数组**的场景。利用数组元素值都在 `[1, n]` 的特性，我们可以将元素值本身作为数组的“下标”进行原地状态标记。遍历数组，当遇到数字 `x` 时，去检查索引为 `abs(x)` 的元素。如果该元素已经是负数，说明之前已经访问过索引 `abs(x)`，即当前访问的 `abs(x)` 就是重复的数字；如果不是负数，则将其变为负数作为已访问的标记。

**C语言实现：**

```c
#include <stdlib.h>

int findDuplicate(int* nums, int numsSize) {
    for (int i = 0; i < numsSize; i++) {
        int val = abs(nums[i]);
        
        // 发现目标位置已经被标记为负数，说明出现了重复
        if (nums[val] < 0) {
            return val;
        }
        
        // 标记目标位置为负数
        nums[val] = -nums[val];
    }
    
    return -1;
}
```

## 核心方法优劣势全景对比

| 算法方法 | 时间复杂度 | 空间复杂度 | 修改原数组 | 优劣势剖析与适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **快慢指针法** | $O(n)$ | $O(1)$ | 否 | **优势：** 降维打击，完美满足题目的所有严苛限制，是理论上的绝对最优解。<br>**劣势：** 思维跳跃极大，如果没有见过这种题型，极难在面试的高压环境下将“数组”与“链表环”联系起来。<br>**场景：** 追求极致性能，且严格限制空间与原数组的终极解法。 |
| **二分查找法** | $O(n \log n)$ | $O(1)$ | 否 | **优势：** 抽屉原理的应用非常符合直觉，思路极其通用，可解释性极强。<br>**劣势：** 时间复杂度相较于线性遍历略高了一点。<br>**场景：** 记不住快慢指针时的最佳保底方案，也是面试官最容易接受的标准解法。 |
| **二进制还原法** | $O(n \log n)$ | $O(1)$ | 否 | **优势：** 避开了对数组元素的直接修改，位运算的角度非常新颖。<br>**劣势：** 时间复杂度受限于常数项（需循环 $32$ 次），且代码量相对繁琐，容易写错边界。<br>**场景：** 适合作为发散思维的补充，用来向面试官炫技或展示扎实的位运算基本功。 |
| **原地哈希法** | $O(n)$ | $O(1)$ | **是** | **优势：** 时空复杂度双双拉满，代码极其精简，逻辑最符合人类直觉。<br>**劣势：** 致命伤是**破坏了原始数据**，直接违反了本题的核心要求。<br>**场景：** 仅适用于**不限制修改数组**的常规找重复题型（例如剑指 Offer 中的类似题目），是解除封印后的首选杀招。 |

**面试实战建议：**
如果在面试中遇到这道题，最稳妥的策略是**循序渐进**：先抛出直观但破坏数组的“原地哈希法”；当面试官施加“不能修改数组”的限制时，给出稳健的“二分查找法”；最后，当面试官追问“能不能把时间压到 $O(n)$”时，再甩出“快慢指针法”完成绝杀。这样的答题节奏能完美展现你的思考深度。
