# 算法学习笔记：LeetCode 74. 搜索二维矩阵 (Search a 2D Matrix)

## 🏷️ 核心标签

`#二分查找 (Binary Search)` `#矩阵 (Matrix)` `#坐标映射 (Index Mapping)` `#时间复杂度优化`

## 原题描述

给你一个满足下述两条性质的 `m x n` 整数矩阵 `matrix`：

* 每行中的整数从左到右按非严格递增顺序排列。
* 每行的第一个整数大于前一行的最后一个整数。

给你一个整数 `target` ，如果 `target` 在矩阵中，返回 `true` ；否则，返回 `false` 。

## 题目剖析

**破题眼（核心条件）：**
结合题目给出的两条性质可以得出一个关键结论：**整个矩阵在逻辑上就是一个严格单调递增的一维数组**。
在算法题中，只要看到“严格单调递增”和“查找”这两个关键字，第一反应就应该是使用**二分查找（Binary Search）**。

## 解法演进与对比

### ❌ 暴力遍历 (Brute Force)

* **思路**：两层 `while` 或 `for` 循环遍历整个矩阵寻找目标值。
* **复杂度**：时间 $O(m \times n)$，空间 $O(1)$。
* **评价**：完全没有利用矩阵的有序特性，属于无效解法。

### ⚠️ 线性搜行 + 二分搜列（常规解法）

* **思路**：
先从上到下扫描第一列（或者对比行首尾），找到 `target` 可能落在哪一**行**。然后在确定的那一行里，使用标准的一维二分查找。
* **复杂度**：时间 $O(m + \log n)$，空间 $O(1)$。
* **评价**：利用了行内的有序性，但没有充分利用“行与行之间也有序”的特性。当矩阵非常狭长（如 $10000 \times 1$）时，寻找行的过程会使算法退化成 $O(N)$ 的线性查找。

### ✅ 全局二分查找（逻辑展开为一维数组 / 最优解）

* **思路**：
不需要真的去额外申请一个一维数组。只需要设定好逻辑上的一维双指针 `left = 0` 和 `right = m * n - 1`，然后在每次求出中间点 `mid` 后，将其**映射回二维矩阵的坐标**即可进行值比对。
* **坐标映射公式**（牢记，矩阵题常用技巧）：
假设矩阵列数为 `n`，一维虚拟索引为 `index`：
* **行坐标 (row)** = `index / n` （整除列数）
* **列坐标 (col)** = `index % n` （对列数取余）


* **复杂度**：时间 $O(\log(m \times n))$，空间 $O(1)$。

## 核心代码模板 (C语言)

```c
#include <stdbool.h>

bool searchMatrix(int** matrix, int matrixSize, int* matrixColSize, int target) {
    int m = matrixSize;
    int n = matrixColSize[0]; 
    
    int left = 0;
    int right = m * n - 1; 

    while (left <= right) {
        // 防溢出的 mid 计算法
        int mid = left + (right - left) / 2;
        
        // 核心：一维索引转二维坐标的映射公式
        int midValue = matrix[mid / n][mid % n];
        
        if (midValue == target) {
            return true;
        } else if (midValue < target) {
            left = mid + 1; // 去右半边找
        } else {
            right = mid - 1; // 去左半边找
        }
    }
    return false;
}

```

## 易错点与细节提醒

* **获取列数的方式：** 在 C 语言中，二维数组传参带有 `matrixColSize` 数组，列数应该用 `matrixColSize[0]` 获取。
* **`mid` 计算防溢出：** 永远养成写 `mid = left + (right - left) / 2` 的习惯。虽然本题节点数量一般不会超过 int 上限，但这是防止整型溢出的良好工程规范。
* **除数与模数：** 坐标映射公式中，`/` 和 `%` 后面的数字**一定是列数 $n$**，千万不要写成行数 $m$！因为矩阵是按行依次排列的，每满一行（即走过 $n$ 个元素），行号才会加 1。

## 举一反三 (拓展延伸)

**如果题目少了第二个条件呢？**
这就变成了另一道经典题：**LeetCode 240. 搜索二维矩阵 II**。
当条件变为“每行递增，每列递增”，但“下一行的首元素不一定大于上一行的尾元素”时，就无法直接用全局一维二分了。此时需要用到**Z字形查找（从右上角或左下角作为起点搜索）**，每次比较可以排除一行或一列，时间复杂度为 $O(m + n)$。这侧面说明了原题中的性质二是能将时间压榨到 $O(\log)$ 级别的关键。
