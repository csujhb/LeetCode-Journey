# 算法学习笔记：LeetCode 33. 搜索旋转排序数组 (Search in Rotated Sorted Array)

## 🏷️ 核心标签

`#二分查找 (Binary Search)` `#局部有序` `#边界条件划分`

## 原题描述

整数数组 `nums` 按升序排列，数组中的值 **互不相同** 。
在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。
例如， `[0,1,2,4,5,6,7]` 在下标 3 处经旋转后可能变为 `[4,5,6,7,0,1,2]`。

给你 **旋转后** 的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1`。
你必须设计一个时间复杂度为 $O(\log n)$ 的算法解决此问题。

## 题目剖析 (破题核心)

这道题是二分查找的进阶天花板之一。常规的二分查找要求数组**完全单调递增**，但这个数组被旋转过一次，变成了**两段各自单调递增的子数组**。

**最核心的定律（破题眼）：**
将一个旋转排序数组从任意位置 `mid` 切开，**必定有一半是完全严格有序的，另一半则包含旋转点（即局部无序）**。

因此，我们的思考路径不应该是“目标值比中间值大还是小”（因为大小关系在旋转数组中被打乱了），而是：

1. **先找规律：哪一半是绝对有序的？**
2. **再看目标：`target` 是否在这个绝对有序的区间范围内？**
3. **做决策：如果在，就在有序的那半边继续找；如果不在，就去无序的那半边找。**

## 标准解法演进

### 正确的二分查找逻辑划分

每次计算出 `mid` 后：

* **情况 A：左半部分 `[left, mid]` 是严格有序的** （判断条件：`nums[left] <= nums[mid]`）
* 接下来检查 `target` 是不是在这个范围内？即 `nums[left] <= target && target < nums[mid]`。
* 如果是：说明目标就在左边，收缩右边界 `right = mid - 1`。
* 如果不是：说明目标只能在右边，收缩左边界 `left = mid + 1`。


* **情况 B：右半部分 `[mid, right]` 是严格有序的** （上面的条件不成立，自然就是右半部分有序）
* 接下来检查 `target` 是不是在这个范围内？即 `nums[mid] < target && target <= nums[right]`。
* 如果是：说明目标就在右边，收缩左边界 `left = mid + 1`。
* 如果不是：说明目标只能在左边，收缩右边界 `right = mid - 1`。



## 核心代码模板 (C语言)

```c
#include <stdbool.h>

int search(int* nums, int numsSize, int target) {
    int left = 0;
    int right = numsSize - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        // 运气好，直接找到
        if (nums[mid] == target) {
            return mid;
        }

        // 判断哪一部分是严格有序的
        if (nums[left] <= nums[mid]) {
            // 左半部分 [left, mid] 严格升序有序
            // 判断 target 是否落在左半边这个有序区间内
            if (target >= nums[left] && target < nums[mid]) {
                right = mid - 1; // 目标在左边，缩小右边界
            } else {
                left = mid + 1;  // 否则，目标肯定在右边包含旋转点的那一半
            }
        } else {
            // 右半部分 [mid, right] 严格升序有序
            // 判断 target 是否落在右半边这个有序区间内
            if (target > nums[mid] && target <= nums[right]) {
                left = mid + 1;  // 目标在右边，缩小左边界
            } else {
                right = mid - 1; // 否则，目标肯定在左边包含旋转点的那一半
            }
        }
    }

    return -1; // 没找到
}

```

## 易错点与细节提醒

* **等号非常重要 (`nums[left] <= nums[mid]`)：**
为什么这里必须是 `<=` 而不是 `<`？因为当 `left` 和 `mid` 指向同一个元素时（例如区间只剩两个元素，求出的 `mid` 总是偏左），`nums[left] == nums[mid]` 是成立的。此时一个元素的区间也应当被视为“有序区间”进行正常逻辑判断，如果不加等号，就会错误地跳入 else 分支导致越界或漏判。
* **提前剪枝是不必要的：**
你在原始代码中写了 `if (numsSize == 1 && nums[0] != target) return -1;`。实际上，良好的 `while(left <= right)` 循环条件能够完美兼容 `numsSize == 1` 的边界情况，保持代码整洁即可，无需单独写出。

## 举一反三 (拓展延伸)

**如果数组中有重复元素怎么办？**
这就引申出了另一道题：**LeetCode 81. 搜索旋转排序数组 II**。
当数组允许重复时，会出现 `nums[left] == nums[mid] == nums[right]` 的极端情况（例如 `[3, 1, 2, 3, 3, 3, 3]`）。此时你完全无法分辨到底哪半边是有序的。
解决办法是：遇到这种情况，只能保守地执行 `left++` 和 `right--` 来一点点缩小区间，时间复杂度在最坏情况下会退化为 $O(N)$。
