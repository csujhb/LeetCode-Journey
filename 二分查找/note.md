# 📝 二分查找进阶笔记：从精确搜索到边界搜索

## 一、 核心题目：LeetCode 35. 搜索插入位置

### 1. 题目描述

给定一个排序数组和一个目标值，返回其索引。如果不存在，返回它将会被按顺序插入的位置。要求时间复杂度为 $O(\log n)$。

### 2. 标准实现 (寻找精确值)

C

```
int searchInsert(int* nums, int numsSize, int target) {
    int left = 0, right = numsSize - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 向下取整
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return left; // 循环结束时，left 即为插入位置
}
```

## 二、 进阶：寻找边界值 (左边界或右边界)

在实际工程中，我们经常需要寻找“第一个”或“最后一个”满足条件的元素。此时使用 `while (left < right)` 模版。

### 1. 寻找边界的应用案例

- **查找元素的范围 (LeetCode 34)**：在包含重复元素的数组中，分别找目标值的第一个（左边界）和最后一个（右边界）位置。
- **第一个错误版本 (LeetCode 278)**：在一系列版本 `[好, 好, 好, 坏, 坏]` 中寻找第一个“坏”掉的版本。
- **整数平方根 (LeetCode 69)**：寻找最后一个满足 $k^2 \le x$ 的整数 $k$。
- **二分答案法 (LeetCode 875/1011)**：在复杂的实际问题中，寻找满足条件的“最小运载能力”或“最慢速度”。

## 三、 🌟 边界搜索的“黄金分支搭配”

在 `while (left < right)` 模版下，分支必须严格成对出现，且 `mid` 的计算必须与分支逻辑配套，否则会陷入死循环。

### 1. 寻找左边界 (第一个满足条件的点)

- **逻辑搭配**：`right = mid` 与 `left = mid + 1`
- **原理**：如果 `mid` 满足条件，它可能是答案，也可能答案在它左边，所以要保留 `mid`（即 `right = mid`）。如果 `mid` 不满足，则彻底排除，即 `left = mid + 1`。
- **配套 mid**：**向下取整** `mid = left + (right - left) / 2`

### 2. 寻找右边界 (最后一个满足条件的点)

- **逻辑搭配**：`left = mid` 与 `right = mid - 1`
- **原理**：如果 `mid` 满足条件，它可能是答案，也可能答案在它右边，所以要保留 `mid`（即 `left = mid`）。如果 `mid` 不满足，则彻底排除，即 `right = mid - 1`。
- **配套 mid**：**向上取整** `mid = left + (right - left + 1) / 2`

------

## 四、 深度解析：为什么寻找精确值时 mid 不讲究？

在“情况 A”（`while (left <= right)`）中，无论是向下取整还是向上取整，**都不会死循环**。

- **根本原因**：因为你的更新逻辑是 `left = mid + 1` 和 `right = mid - 1`。只要 `mid` 不是目标值，搜索区间就一定会**强制性地至少收缩 1 个单位**。只要区间一直在缩小，循环就一定会终止。
- **工程习惯**：在 C 语言中，整数除法默认向下取整。直接写 `mid = left + (right - left) / 2` 最简洁，符合程序员的“肌肉记忆”。

## 五、 复杂度与避坑小结

| **场景**       | **循环条件**    | **分支逻辑**                     | **mid 计算** | **记忆重点**                        |
| -------------- | --------------- | -------------------------------- | ------------ | ----------------------------------- |
| **精确搜索**   | `left <= right` | `+1` / `-1` 均跳过 mid           | 向下取整     | 无脑向下取整即可                    |
| **左边界搜索** | `left < right`  | `right = mid` / `left = mid + 1` | 向下取整     | **right = mid** 配 **向下取整**     |
| **右边界搜索** | `left < right`  | `left = mid` / `right = mid - 1` | 向上取整     | **left = mid** 配 **向上取整 (+1)** |

- **防死循环口诀**：有加（`left = mid`）必有加（`mid` 计算 +1）。
- **整型溢出**：务必使用 `left + (right - left) / 2` 代替 `(left + right) / 2`。

是的，你的直觉非常敏锐！在 **`while (left < right)`** 的模版中，分支通常是成对出现的。

为了确保搜索空间不断收缩，且不漏掉目标边界，分支的组合逻辑只有以下两种固定的“黄金搭配”：

---

### 1. 寻找左边界（第一个满足条件的元素）
当你的逻辑是 **`right = mid`** 时，另一个分支**必须**是 **`left = mid + 1`**。

* **逻辑原理**：
    * 如果 `mid` 满足条件（比如 `nums[mid] >= target`），那么 `mid` 有可能是我们要找的第一个位置，也可能它右边还有更早的位置。所以我们要保留 `mid`，即 `right = mid`。
    * 如果 `mid` 不满足条件，说明 `mid` 及其左边的所有元素都绝对不可能是答案。所以我们要彻底跳过 `mid`，即 `left = mid + 1`。
* **配套 mid 公式**：向下取整 `left + (right - left) / 2`。



---

### 2. 寻找右边界（最后一个满足条件的元素）
当你的逻辑是 **`left = mid`** 时，另一个分支**必须**是 **`right = mid - 1`**。

* **逻辑原理**：
    * 如果 `mid` 满足条件（比如 `nums[mid] <= target`），那么 `mid` 有可能是我们要找的最后一个位置，也可能它右边还有更晚的位置。所以我们要保留 `mid`，即 `left = mid`。
    * 如果 `mid` 不满足条件，说明 `mid` 及其右边的所有元素都绝对不可能是答案。所以我们要彻底跳过 `mid`，即 `right = mid - 1`。
* **配套 mid 公式**：向上取整 `left + (right - left + 1) / 2`。



---

### 💡 为什么不能像“找精确值”那样写成 `right = mid - 1` 和 `left = mid + 1`？

如果你在 `while (left < right)` 的情况下把两个分支都写成跳过 `mid`，会发生非常危险的事：**你会把唯一的正确答案给“跳”过去。**

**举个例子：**
数组 `[1, 5, 10]`，找第一个大于等于 `5` 的数。
1.  `left = 0, right = 2, mid = 1`。
2.  `nums[1]` 是 `5`，满足条件。
3.  如果你执行 `right = mid - 1`，那么 `right` 变成了 `0`。
4.  现在区间变成了 `[0, 0]`，循环结束。
5.  **结果**：你错过了索引 `1`（数字 5），返回了一个错误答案。

**总结：**
在寻找边界时，满足条件的那个 `mid` **可能是最终答案**，所以我们必须用 `right = mid`（或 `left = mid`）把它“圈”在搜索区间里，直到 `left` 和 `right` 最终重合在那一点上。

### 记忆终极口诀：
* **有加必有减**：一个分支不加 1（`right = mid`），另一个分支就必须加 1（`left = mid + 1`）。
* **`mid` 跟着不加 1 的跑**：
    * 如果 `left` 分支是 `left = mid`（不加 1），那么计算 `mid` 的时候就要手动 **+1**（向上取整）。
    * 如果 `right` 分支是 `right = mid`（不加 1），那么计算 `mid` 的时候就 **不加 1**（向下取整）。
