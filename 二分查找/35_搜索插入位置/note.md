# 📝 LeetCode 35. 搜索插入位置 学习笔记

## 题目描述

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 $O(\log n)$ 的算法。

**示例 1:**
> **输入:** nums = [1,3,5,6], target = 5
> **输出:** 2

**示例 2:**
> **输入:** nums = [1,3,5,6], target = 2
> **输出:** 1

**示例 3:**
> **输入:** nums = [1,3,5,6], target = 7
> **输出:** 4

---

## 核心思想：二分查找 (Binary Search)

这道题是学习二分查找最经典的入门题。题目有两个关键线索：
* **有序数组**：这是使用二分查找的前提条件。
* **$O(\log n)$ 时间复杂度**：这是二分查找的标志性特征。

**关键结论：**
当二分查找循环 `while (left <= right)` 正常结束（即未找到目标值）时，`left` 指针所在的位置，正好就是该目标值应该按顺序插入的位置。



---

## C 语言标准实现代码

```c
int searchInsert(int* nums, int numsSize, int target) {
    int left = 0;
    int right = numsSize - 1;

    while (left <= right) {
        // 防止整型溢出的 mid 计算方式（向下取整）
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid; // 找到目标值，直接返回索引
        } else if (nums[mid] < target) {
            left = mid + 1; // 目标值在右半部分
        } else {
            right = mid - 1; // 目标值在左半部分
        }
    }

    // 循环结束未找到，left 记录的就是插入位置
    return left;
}
```

## 🌟 额外加餐：二分查找“生死判别法”

二分查找最难的地方在于细节：什么时候用 `<=`？什么时候 `mid` 要加 1？我们可以通过以下逻辑彻底理清。

### 判定一：循环条件的逻辑选择

**情况 A：寻找精确值（如本题）**
* **特征**：使用 `while (left <= right)`。
* **含义**：搜索区间是**闭区间** `[left, right]`。
* **退出**：当 `left > right` 时区间为空，完全找完了。
* **配套**：`right = mid - 1` 和 `left = mid + 1`（因为 `mid` 已经检查过不是目标了，直接剔除）。

**情况 B：寻找边界值（左边界或右边界）**
* **特征**：使用 `while (left < right)`。
* **含义**：寻找满足某种条件的第一个元素，当 `left == right` 时，我们就找到了那个唯一的答案。
* **配套**：通常会有 `right = mid` 或 `left = mid` 的操作，即保留 `mid` 作为潜在答案。

### 判定二：避免死循环的 `mid` 公式

当你使用 `while (left < right)` 且需要保留 `mid` 时，必须考虑“死循环”问题。这通常发生在搜索区间只剩下 **2 个元素**时。



**防死循环口诀：**

* 如果你的收缩逻辑是 **`right = mid`**：
    * **规则**：使用**向下取整** `mid = left + (right - left) / 2`。
    * 原理：只有这样才能让 right 往左缩，最终使 left == right 退出循环。
* 如果你的收缩逻辑是 **`left = mid`**：
    * **规则**：使用**向上取整** `mid = left + (right - left + 1) / 2`。
    * 原理：只有这样才能让 left 往右跳，避免始终停在原地的死循环。

---

## 🔍 深度解析：寻找精确值时 mid 计算的“讲究”

在“情况 A”（`while (left <= right)`）中，你可能会问：使用向下取整还是向上取整（即加不加那个 1）有区别吗？

**结论是：在寻找精确值的场景下，两者皆可，都不会死循环。**

* **为什么都不会死循环？**
因为你的更新逻辑是 `left = mid + 1` 和 `right = mid - 1`。只要 `mid` 不是目标值，搜索区间就一定会强制性地至少收缩 1 个单位。只要区间在缩小，循环就一定会终止。

* **那有什么区别？**
    * **行业标准**：在 C 语言中，整数除法 `/` 默认就是向下取整。直接写 `mid = left + (right - left) / 2` 最简洁，符合全世界程序员的“肌肉记忆”。
    * **比较顺序**：当剩余偶数个元素（如剩下 2 个）时，向下取整会先检查左边那个，向上取整会先检查右边那个。它们只是“殊途同归”，内部比较顺序略有不同，但不影响结果。
    * **最佳实践**：为了减少脑细胞损耗，建议在寻找精确值时**无脑选择向下取整**。将对 `+1` 的警觉性全部留给寻找边界的场景。

---

## 复杂度分析

* **时间复杂度**：$O(\log n)$。由于每次将搜索范围缩小一半，查找 $n$ 个元素最多只需要进行 $\log_2 n$ 次比较。
* **空间复杂度**：$O(1)$。只使用了 `left`, `right`, `mid` 三个辅助变量。

---

## 避坑指南

* **整型溢出**：永远不要写 `mid = (left + right) / 2`，因为 `left + right` 可能会超过 `int` 的最大值。请务必使用 `mid = left + (right - left) / 2`。
* **插入位置**：如果数组中所有元素都小于目标值，`left` 会一直右移直到等于 `numsSize`，这正好是数组末尾的插入位置。
